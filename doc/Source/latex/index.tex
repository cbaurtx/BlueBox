\item Basic Example\+: 
\end{DoxyItemize}\hypertarget{index_autotoc_md603}{}\doxysection{\texorpdfstring{Example Brief}{Example Brief}}\label{index_autotoc_md603}
This A2\+DP SINK example uses the Classic Bluetooth A2\+DP profile to distribute and receive audio streams and the AVRCP profile to notify and control media information. Applications such as Bluetooth speakers or Bluetooth headsets can use this example as a reference for basic functions.

The complete pipeline of A2\+DP SINK is as follows\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{aadp\_source\ -\/-\/-\/>\ aadp\_sink\_stream\ -\/-\/-\/>\ i2s\_stream\_writer\ -\/-\/-\/>\ codec\_chip\ -\/-\/-\/>\ speaker}

\end{DoxyCode}


As A2\+DP SINK role, the device in this example needs to be paired and connected with A2\+DP SOURCE (usually a mobile phone) in order to play the audio distributed by the SOURCE.\hypertarget{index_autotoc_md604}{}\doxysection{\texorpdfstring{Environment Setup}{Environment Setup}}\label{index_autotoc_md604}
\hypertarget{index_autotoc_md605}{}\doxysubsubsection{\texorpdfstring{Hardware Required}{Hardware Required}}\label{index_autotoc_md605}
This example runs on the boards that are marked with a green checkbox in the \href{../../README.md\#compatibility-of-examples-with-espressif-audio-boards}{\texttt{table}}. Please remember to select the board in menuconfig as discussed in Section Configuration below.\hypertarget{index_autotoc_md606}{}\doxysection{\texorpdfstring{Build and Flash}{Build and Flash}}\label{index_autotoc_md606}
\hypertarget{index_autotoc_md607}{}\doxysubsection{\texorpdfstring{Default IDF Branch}{Default IDF Branch}}\label{index_autotoc_md607}
This example supports IDF release/v5.\+0 and later branches. By default, it runs on ADF\textquotesingle{}s built-\/in branch {\ttfamily \$ADF\+\_\+\+PATH/esp-\/idf}.\hypertarget{index_autotoc_md608}{}\doxysubsection{\texorpdfstring{Configuration}{Configuration}}\label{index_autotoc_md608}
The default board for this example is {\ttfamily ESP32-\/\+Lyrat V4.\+3}, if you need to run this example on other development boards, select the board in menuconfig, such as {\ttfamily ESP32-\/\+Lyrat-\/\+Mini V1.\+1}.


\begin{DoxyCode}{0}
\DoxyCodeLine{menuconfig\ >\ Audio\ HAL\ >\ ESP32-\/Lyrat-\/Mini\ V1.1}

\end{DoxyCode}
\hypertarget{index_autotoc_md609}{}\doxysubsection{\texorpdfstring{Build and Flash}{Build and Flash}}\label{index_autotoc_md609}
Build the project and flash it to the board, then run monitor tool to view serial output (replace {\ttfamily PORT} with your board\textquotesingle{}s serial port name)\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{idf.py\ -\/p\ PORT\ flash\ monitor}

\end{DoxyCode}


To exit the serial monitor, type {\ttfamily Ctrl-\/\mbox{]}}.

See \href{https://docs.espressif.com/projects/esp-idf/en/release-v5.3/esp32/index.html}{\texttt{ESP-\/\+IDF Programming Guide}} for full steps to configure and build an ESP-\/\+IDF project.\hypertarget{index_autotoc_md610}{}\doxysection{\texorpdfstring{How to Use the Example}{How to Use the Example}}\label{index_autotoc_md610}

\hypertarget{index_autotoc_md613}{}\doxysection{\texorpdfstring{Troubleshooting}{Troubleshooting}}\label{index_autotoc_md613}

\begin{DoxyItemize}
\item ESP32 A2\+DP supports the mandatory audio codec SBC. The SBC data stream is transmitted to A2\+DP SINK, and then decoded into PCM data for output. The PCM data is usually 44.\+1 k\+Hz sampling rate, dual-\/channel, and 16-\/bit sampling data. ESP32 A2\+DP SINK also supports other decoder configuration, but you need to modify the protocol stack settings.
\item ESP32 A2\+DP SINK can only connect to one remote A2\+DP SOURCE device at most. In addition, A2\+DP SINK cannot be used with A2\+DP SOURCE at the same time, but it can be used with other configuration files, such as SPP and HFP.
