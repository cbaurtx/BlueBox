Implementers of their own bigints should copy below this directory to another directory and do the following\+:


\begin{DoxyItemize}
\item Rewrite {\ttfamily spec.\+author}, {\ttfamily spec.\+license}, {\ttfamily spec.\+homepage} and {\ttfamily spec.\+summary} in {\ttfamily \texorpdfstring{$<$}{<}gem-\/dir\texorpdfstring{$>$}{>}/mrbgem.rake} file to those of your own implementers.
\item Implement the respective functions in {\ttfamily \texorpdfstring{$<$}{<}gem-\/dir\texorpdfstring{$>$}{>}/core/bigint.c}.
\begin{DoxyItemize}
\item Define and use an object structure for {\ttfamily MRB\+\_\+\+TT\+\_\+\+BIGINT} type-\/tag. It is recommended to use {\ttfamily mrb\+\_\+static\+\_\+assert\+\_\+object\+\_\+size()} to ensure that the size of the object structure is within six words.
\end{DoxyItemize}
\item Delete this file from the destination of the copy.
\end{DoxyItemize}

If you wish to use it as an alternative to the {\ttfamily mruby-\/bigint} provided by mruby, please leave the GEM name in {\ttfamily \texorpdfstring{$<$}{<}gem-\/dir\texorpdfstring{$>$}{>}/mrbgem.rake} as it is. This is an important factor when it is depended from other GEMs with {\ttfamily spec.\+add\+\_\+dependency \textquotesingle{}mruby-\/bigint\textquotesingle{}}.

The name of the top directory of the GEM can be changed arbitrarily. The name of the Git repository can also be changed arbitrarily.

Note that there is no need for an initialization function as there is in a normal GEM. If you need it, create a file {\ttfamily \texorpdfstring{$<$}{<}gem-\/dir\texorpdfstring{$>$}{>}/src/bigint.c} for example, and implement the {\ttfamily mrb\+\_\+mruby\+\_\+bigint\+\_\+gem\+\_\+init()} function.
