\doxysubsection{Detailed Description}
\hypertarget{dir_7321c74b352824205bb37dc7cc56086f_autotoc_md110}{}\doxysubsection{\texorpdfstring{ESP-\/\+DSP ESP32-\/\+Azure IoT kit 3d graphics demo application}{ESP-\/\+DSP ESP32-\/\+Azure IoT kit 3d graphics demo application}}\label{dir_7321c74b352824205bb37dc7cc56086f_autotoc_md110}
The demo is developed for \href{https://github.com/espressif/esp-bsp/tree/master/esp32_azure_iot_kit}{\texttt{ESP32-\/\+Azure IoT kit}} development board and is demonstrating the usage of matrices with {\ttfamily ESP-\/\+DSP} {\ttfamily Mat} class, Kalman filter and basic 3D graphics.

The 3D Graphics demo displays a 2D graphics, converted to 3D as a 3D rotating object, on the development board\textquotesingle{}s display. Button press changes the rotation direction of the 3D object. Run the menuconfig using the following command\+: \begin{DoxyVerb}idf.py mencuonfig
\end{DoxyVerb}


In the menuconfig\textquotesingle{}s menu item {\ttfamily Demo user configuration} select which 3D object to display. It\textquotesingle{}s either a 3D cube, or ESP logo, or a user-\/defined graphics. Getting the user-\/defined graphics is described in an \href{../../graphics/img_to_3d_matrix/example/}{\texttt{example}}\hypertarget{dir_7321c74b352824205bb37dc7cc56086f_autotoc_md111}{}\doxysubsection{\texorpdfstring{Running the demo}{Running the demo}}\label{dir_7321c74b352824205bb37dc7cc56086f_autotoc_md111}
To start the demo, run the following command\+: \begin{DoxyVerb}idf.py build flash monitor
\end{DoxyVerb}


The expected output is the following\+: \begin{DoxyVerb}I (570) 3D image demo: Selected 3D image - ESP Logo
I (570) 3D image demo: Showing ESP text
I (6730) 3D image demo: Showing 3D image
\end{DoxyVerb}


Note, that the first line {\ttfamily Selected 3D image} from the expected output depends on the user\textquotesingle{}s Kconfing menu selection
