\doxysubsection{Detailed Description}
\hypertarget{dir_e1fd1074c74aa6c4545f8cfae2dc8758_autotoc_md579}{}\doxysubsection{\texorpdfstring{µnit}{µnit}}\label{dir_e1fd1074c74aa6c4545f8cfae2dc8758_autotoc_md579}
µnit is a small but full-\/featured unit testing framework for C. It has no dependencies (beyond libc), is permissively licensed (MIT), and is easy to include into any project.

For more information, see \href{https://nemequ.github.io/munit}{\texttt{the µnit web site}}.

\href{https://travis-ci.org/nemequ/munit}{\texttt{}} \href{https://ci.appveyor.com/project/quixdb/munit/branch/master}{\texttt{}}\hypertarget{dir_e1fd1074c74aa6c4545f8cfae2dc8758_autotoc_md580}{}\doxysubsection{\texorpdfstring{Features}{Features}}\label{dir_e1fd1074c74aa6c4545f8cfae2dc8758_autotoc_md580}
Features µnit currently includes include\+:


\begin{DoxyItemize}
\item Handy assertion macros which make for nice error messages.
\item Reproducible cross-\/platform random number generation, including support for supplying a seed via CLI.
\item Timing of both wall-\/clock and CPU time.
\item Parameterized tests.
\item Nested test suites.
\item Flexible CLI.
\item Forking (\href{https://github.com/nemequ/munit/issues/2}{\texttt{except on Windows}}).
\item Hiding output of successful tests.
\end{DoxyItemize}

Features µnit does not currently include, but some day may include (a.\+k.\+a., if you file a PR…), include\+:


\begin{DoxyItemize}
\item \href{http://testanything.org/}{\texttt{TAP}} support; feel free to discuss in \href{https://github.com/nemequ/munit/issues/1}{\texttt{issue \#1}}
\end{DoxyItemize}\hypertarget{dir_e1fd1074c74aa6c4545f8cfae2dc8758_autotoc_md581}{}\doxysubsubsection{\texorpdfstring{Include into your project with meson}{Include into your project with meson}}\label{dir_e1fd1074c74aa6c4545f8cfae2dc8758_autotoc_md581}
