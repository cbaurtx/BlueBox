\hypertarget{md_managed__components_2espressif____nghttp_2nghttp2_2third-party_2mruby_2doc_2mruby3_81_autotoc_md474}{}\doxysubsubsection{\texorpdfstring{{\ttfamily OP\+\_\+\+SEND} and {\ttfamily OP\+\_\+\+SENDB}}{{\ttfamily OP\+\_\+\+SEND} and {\ttfamily OP\+\_\+\+SENDB}}}\label{md_managed__components_2espressif____nghttp_2nghttp2_2third-party_2mruby_2doc_2mruby3_81_autotoc_md474}
Method calling instructions are unified. Now {\ttfamily OP\+\_\+\+SEND} and {\ttfamily OP\+\_\+\+SENDB} (method call with a block) can support both splat arguments and keyword arguments as well.

The brief description of the instructions\+:

\texorpdfstring{$\vert$}{|}{\ttfamily OP\+\_\+\+SEND} \texorpdfstring{$\vert$}{|} BBB \texorpdfstring{$\vert$}{|} {\ttfamily R\mbox{[}a\mbox{]} = R\mbox{[}a\mbox{]}.call(Syms\mbox{[}b\mbox{]},R\mbox{[}a+1..n\mbox{]},R\mbox{[}a+n+1\mbox{]},R\mbox{[}a+n+2\mbox{]}..nk) c=n\texorpdfstring{$\vert$}{|}nk\texorpdfstring{$<$}{<}\texorpdfstring{$<$}{<}4} \texorpdfstring{$\vert$}{|} \texorpdfstring{$\vert$}{|}{\ttfamily OP\+\_\+\+SENDB} \texorpdfstring{$\vert$}{|} BBB \texorpdfstring{$\vert$}{|} {\ttfamily R\mbox{[}a\mbox{]} = R\mbox{[}a\mbox{]}.call(Syms\mbox{[}b\mbox{]},R\mbox{[}a+1..n\mbox{]},R\mbox{[}a+n+1..nk\mbox{]},R\mbox{[}a+n+2..nk\mbox{]},\&R\mbox{[}a+n+2\texorpdfstring{$\ast$}{*}nk+2\mbox{]}) c=n\texorpdfstring{$\vert$}{|}nk\texorpdfstring{$<$}{<}\texorpdfstring{$<$}{<}4} \texorpdfstring{$\vert$}{|}

Operand C specifies the number of arguments. Lower 4 bits ({\ttfamily n}) represents the number of ordinal arguments, and higher 4 bits ({\ttfamily nk}) represents the number of keyword arguments. When {\ttfamily n == 15}, the method takes arguments packed in an array. When {\ttfamily nk == 15}, the method takes keyword arguments are packed in a hash.\hypertarget{md_managed__components_2espressif____nghttp_2nghttp2_2third-party_2mruby_2doc_2mruby3_81_autotoc_md475}{}\doxysubsubsection{\texorpdfstring{{\ttfamily OP\+\_\+\+ARYPUSH}}{{\ttfamily OP\+\_\+\+ARYPUSH}}}\label{md_managed__components_2espressif____nghttp_2nghttp2_2third-party_2mruby_2doc_2mruby3_81_autotoc_md475}
Now takes 2 operands and pushes multiple entries to an array.\hypertarget{md_managed__components_2espressif____nghttp_2nghttp2_2third-party_2mruby_2doc_2mruby3_81_autotoc_md476}{}\doxysubsection{\texorpdfstring{Boxing Updated}{Boxing Updated}}\label{md_managed__components_2espressif____nghttp_2nghttp2_2third-party_2mruby_2doc_2mruby3_81_autotoc_md476}
\hypertarget{md_managed__components_2espressif____nghttp_2nghttp2_2third-party_2mruby_2doc_2mruby3_81_autotoc_md477}{}\doxysubsubsection{\texorpdfstring{Word Boxing}{Word Boxing}}\label{md_managed__components_2espressif____nghttp_2nghttp2_2third-party_2mruby_2doc_2mruby3_81_autotoc_md477}
{\ttfamily MRB\+\_\+\+WORD\+\_\+\+BOXING} now packs floating-\/point numbers in the word, if the size of {\ttfamily mrb\+\_\+float} is equal or smaller than the size of {\ttfamily mrb\+\_\+int} by default. If the size of {\ttfamily mrb\+\_\+float} and {\ttfamily mrb\+\_\+int} are same, the last 2 bits in the {\ttfamily mrb\+\_\+float} are trimmed and used as flags. If you need full precision, you need to define {\ttfamily MRB\+\_\+\+WORDBOX\+\_\+\+NO\+\_\+\+FLOAT\+\_\+\+TRUNCATE} as described above.\hypertarget{md_managed__components_2espressif____nghttp_2nghttp2_2third-party_2mruby_2doc_2mruby3_81_autotoc_md478}{}\doxysubsubsection{\texorpdfstring{NaN Boxing}{NaN Boxing}}\label{md_managed__components_2espressif____nghttp_2nghttp2_2third-party_2mruby_2doc_2mruby3_81_autotoc_md478}
Previous NaN boxing packs values in NaN representation, but pointer retrievals are far more frequent than floating-\/point number references. So we add constant offset to NaN representation to clear higher bits of pointer representation. This representation is called "{}\+Favor Pointer"{} NaN Boxing.

Also, previous NaN boxing limit the size of {\ttfamily mrb\+\_\+int} to 4 bytes (32 bits) to fit in NaN values. Now we allocate integer values in the heap, if the value does not fit in the 32 bit range, just like we did in Word Boxing.\hypertarget{md_managed__components_2espressif____nghttp_2nghttp2_2third-party_2mruby_2doc_2mruby3_81_autotoc_md479}{}\doxysubsection{\texorpdfstring{Constant Folding}{Constant Folding}}\label{md_managed__components_2espressif____nghttp_2nghttp2_2third-party_2mruby_2doc_2mruby3_81_autotoc_md479}
The code generator was updated to reduce the number of instructions, e.\+g.


\begin{DoxyCode}{0}
\DoxyCodeLine{a\ =\ 2\ *\ 5}

\end{DoxyCode}


will be interpreted as


\begin{DoxyCode}{0}

\end{DoxyCode}

\begin{DoxyCode}{0}

\DoxyCodeLine{MOVE\ R1\ R4}

\end{DoxyCode}


to


\begin{DoxyCode}{0}
\DoxyCodeLine{GETIV\ R1\ :@foo}

\end{DoxyCode}
