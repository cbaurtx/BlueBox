FGMP is really in the public domain. You can do whatever you want with it.

I wrote FGMP so that we would all have access to a (truly free) implementation of this subset of the API of GNU libgmp. I encourage everyone to distribute this as widely as possible.

If you need more documentation, I suggest you look at the file gmp.\+texi which is included with the GNU gmp library.

You can send me bug reports, implementations of missing functions, flames and rants by Email.

Any submissions of new code to be integrated into fgmp must also be placed in the public domain (For the particularly dense, you can release a new fgmp yourself under different licensing terms. This is a condition for including a submission in a release of FGMP that I personally prepare).

\item fgmp does not implement the following\+:

\item internal mpn\+\_\+\texorpdfstring{$\ast$}{*} functions
\item mpz\+\_\+perfect\+\_\+square\+\_\+p
\item mpz\+\_\+inp\+\_\+raw, mpz\+\_\+out\+\_\+raw
\item mp\+\_\+set\+\_\+memory\+\_\+functions, mpz\+\_\+out\+\_\+str, mpz\+\_\+inp\+\_\+str
\end{DoxyItemize}
\item fgmp implements the following in addition to the routines in GNU gmp. {\ttfamily int mpz\+\_\+jacobi(\+MP\+\_\+\+INT \texorpdfstring{$\ast$}{*}a, MP\+\_\+\+INT \texorpdfstring{$\ast$}{*}b)}
\begin{DoxyItemize}
\item finds the jacobi symbol (a/b)
\end{DoxyItemize}
\item mpz\+\_\+sizeinbase often overestimates the exact value
\item To convert your gmp based program to fgmp (subject to the above)
\end{DoxyEnumerate}
\begin{DoxyItemize}
\item recompile your source. Make sure to include the gmp.\+h file included with fgmp rather than that included with gmp. (The point is to recompile all files which include gmp.\+h)
\item link with gmp.\+o instead of libgmp.\+a
\end{DoxyItemize}
